O Funcionamento da Internet

Medidas de Grandeza

O computador funciona de forma binaria, interpreta apenas duas ondas: 0 e 1.

Para a medida de grandeza são usaos as seguintes unidades:

  1. bit - menor unidade de medida que um computador possível (armazena 0 ou 1);
  2. Byte - O byte é a junção de 8 bits, formando assim uma sequência de 8 valores que podem ser 1 ou 0;
  3. Kilobyte (KB) - 1.000 bytes;
  4. Megabyte (MB) - 1.000.000 bytes ou então 8.388.608 bits;
  5. Gigabyte (GB) - 1.000.000.000 bytes ou então 8.589.934.592 bits;
  6. Terabyte (TB) - 1.000.000.000.000 bytes ou então 8.796.093.022.208 bits;
  7. Petabyte (PB) - 1.000.000.000.000.000 bytes ou então 9.007.199.254.740.992 bits;

Como fazer a conversão de bytes para bits?

A conversão de bytes para bits é feita de maneira muito simples: basta que você multiplique o valor de seus bytes por 8. Esse cálculo é possível porque cada byte equivale a exatamente 8 bits, sendo assim, para o valor de 512 bytes, temos: 512 * 8 = 4096 bits

A Velocidade da Internet

A velocidade da internet é medida em bits, mais especificamente em bits por segundo. O motivo para isso é que o valor calculado em bits é muito mais atrativo do que em bytes, por exemplo: um anúncio de internet com 100 “megas” é de 100 megabits por segundo, que convertido para bytes se torna 12.5 megabytes por segundo, um valor 8 vezes menor do que o anunciado.

Além disso, diferentemente das medidas de armazenamento, a velocidade de conexão com a internet não necessita de valores extraordinariamente grandes, como gigabits, terabits ou até petabits

A internet vem do mar

A maior parte da comunicação da internet acontece através de cabeamento subáguatico, pouca parte da rede é feita através de satélites, pois eles possuem uma limitação de tráfego e sofrem muito com interferências, e é por isso que os cabos de fibra ótica devem ser passados pelos oceanos. Um trabalho incalculável, mas necessário.

Mas não dá para esses cabos submarinos saírem pela praia e seguirem caminho até a sua casa, e por isso precisam se interligar a outros sistemas de comunicação. Alguns dos sistemas utilizados sempre foram a telefonia tradicional, os sistemas de TV a cabo, os sinais via satélite e até as simples redes de radiofrequência.

O problema é que os sistemas diferentes transmitem sinais em formatos diferentes (ANALÓGICOS E DIGITAIS). Isso dificultaria a comunicação entre pontos, se não fosse um processo de “conversão”, mais conhecido como MODULAÇÃO.

Uma das funções desse aparelho é MODULAR os sinais que saem e DEMODULAR os sinais que chegam. E é por isso que chamamos esse aparelho de MODEM.

Os roteadores (do provedor de internet, não o que GATEWAY/MODEM que fica em sua casa) são os responsáveis por escolheram as rotas que os dados irão tomar para irem do cliente ao servidor ou vice-versa.

Endereço de IP

Internet funciona baseada em um conjunto de protocolos chamado TCP/IP. Quando você “se conecta” à Internet, recebe uma identificação única. Essa identificação é um ENDEREÇO IP.

Os IPs mais antigos (IPv4) usam 4 octetos, que são conjuntos de 8 bits separados por pontos, totalizando 32 bits por identificador.
Ex: 123.45.67.89 = 01111011.00101101.01000011.01011001
Os IPs mais modernos (IPv6), usam 128 bits ao todo (o que é 4x mais bits que o IPv4).
Ex: 2001:0db8:85a3:08d3:1319:8a2e:0370:7344

Acesso aos Servidores

DOMAIN NAME SYSTEM (DNS), serve para armazenar as URL para fazer o link entre o IP do cliente e o do IP do servidor. (IP mudam constantemente e são díficeis de decorar por isso a necessecidade de DNS)

Domínio e Hospedagem

O domínio é um nome único que vai conseguir identificar o seu servidor ou as suas páginas. Para conseguir um domínio, você deve pagar pelo direito de usá-lo por um período mínimo de 1 ano. (Ex: faetec.rj.gov.br; github.io)

TOP LEVEL DOMAIN (TLD) é terminação do domínio, como por exemplo: .com, .or, .io...

GTLG - São TLDs genéricos, sem indicação de país. Alguns dos domínios genéricos são .com, .net, .gov, .org, .io, .info, .online, .store, etc.

ccTLD: São TLDs com designação do país (coutry code). Alguns dos domínios desse tipo são .com.br, .edu.us, .co.fr, .jp, .es, etc.

Sub-Domínio é o que vemantes do domínio. Podem ser vários, por exemplo: www. images. maps. mail. etc.

Além do domínio e sub-domínio a URL também é composta por um protocolo, que pode ser HTTP:// ou HTTPS://, dependendo se o seu servidor tem ou não segurança por SSL (serviço de criptografia de dados).

E por fim, a URl também pode ter um caminho extra, que indica pastas a percorrer para achar um arquivo específico.(após a / que vem logo apos o TLD)

Hospedagem
